Ingest a file, raw text, or a URL. Each call returns an `IngestResult` and is
idempotent by content hash â€” re-ingesting an unchanged `document_id` does no work.

```python
from citenexus.extract.types import SourceType

rag.ingest("report.pdf")                          # a file â†’ extractor picked by extension
rag.ingest(text="Clause 4: 90 days notice.", document_id="c4")   # raw text (plain)
rag.ingest("https://example.com/policy")          # a URL â†’ fetched, HTML-extracted
rag.ingest(image_bytes, source_type=SourceType.image, document_id="fig-1")  # an image
```

Each `ingest()` returns an `IngestResult`:

```python
r = rag.ingest("report.pdf")
print(r.status, r.n_units)     # "ingested" 42   (or "unchanged" 0 on a re-ingest)
print(r.document_id, r.eu_ids) # the id and the Evidence-Unit ids created
```

<Aside type="note" title="One source at a time">
`ingest()` takes a single source. It does **not** walk a directory and there is
**no** no-argument "ingest the whole bucket/folder" call â€” pass each file, URL, or
text yourself. For whole-site intake, use `crawl()` below.
</Aside>

Ingest has an inverse: [`rag.delete(document_id)`](/citenexus/revoke/) (alias
`revoke`) surgically removes one document and everything derived from it â€”
idempotently, and guarding raw blobs shared by identical bytes.

## Crawl a site

`crawl()` does a same-domain breadth-first walk, ingesting each page as HTML:

```python
results = rag.crawl("https://docs.example.com", max_pages=50, max_depth=3)
print(len(results), "pages ingested")
```

## Supported formats

Extraction dispatches by explicit `source_type` first, then file extension, then
falls back to plain text:

| Extension | Extractor |
|---|---|
| `.pdf` | PDF (text + figures, with page/bbox) |
| `.docx` `.pptx` `.xlsx` | Office (OOXML); tables become GFM Markdown |
| `.html` `.htm` | HTML |
| `.md` `.markdown` | Markdown |
| `.csv` | CSV |
| `.txt` | Plain text |
| *anything else* | Plain-text fallback â€” never a hard failure |

<Aside type="caution" title="Images and OCR â€” the honest version">
Standalone image files are **not** routed by extension (`.png`/`.jpg` fall through
to plain text). To ingest an image, pass `source_type=SourceType.image` explicitly.
Figures *inside* PDFs/DOCX/PPTX are handled by the injected [vision](/citenexus/vision/)
model during ingest. CiteNexus ships **no OCR engine** â€” text inside a scanned
image is only recovered if your vision model returns it.
</Aside>

Extraction runs in the shared Rust core, so a PDF or XLSX extracts byte-identically
whether you call from Go, JavaScript, or Python. Which indexes get built from the
extracted units is governed by the [signals](/citenexus/signals/) you declared.
